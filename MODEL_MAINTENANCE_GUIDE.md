# 模型和API管理维护指南

## 📌 概述

本指南说明如何通过前端页面维护模型的URL、API密钥和配置信息。

## 🚀 快速访问

### 访问模型管理页面

**方式1：在主页导航点击**
1. 打开应用主页
2. 左侧导航栏点击 **"模型管理"**
3. 进入模型管理页面

**方式2：直接访问URL**
```
http://localhost:5173/models
```

---

## 📊 模型管理页面功能

### 页面概览

模型管理页面分为两部分：
1. **模型列表** - 显示所有可用的AI模型
2. **操作按钮** - 用于管理每个模型

### 模型列表显示内容

| 列 | 说明 |
|----|------|
| **模型名称** | 模型的显示名称（如：DALL-E 3） |
| **提供商** | 所属提供商（OpenAI、Stability AI等） |
| **API密钥** | 已配置的API密钥数量 |
| **配置项** | 已设置的配置项数量 |
| **操作** | API密钥管理、模型配置按钮 |

## 🔑 API密钥管理

### 添加API密钥

**步骤：**
1. 在模型列表中找到目标模型
2. 点击 **"API密钥"** 按钮
3. 在弹出框中点击 **"添加API密钥"** 按钮
4. 填写以下信息：

#### 表单字段说明

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| **API密钥** | 从AI服务提供商获得的密钥 | `sk-xxx...` | ✅ 是 |
| **配额限制** | 最多可以调用多少次（可选） | `1000` | ❌ 否 |
| **过期时间** | 密钥何时停止工作（可选） | `2025-12-31` | ❌ 否 |

### 查看和管理API密钥

#### 查看密钥详情
1. 在模型的"API密钥管理"页面
2. 可以看到所有已配置的密钥列表
3. 点击 👁️ 图标查看完整的API密钥

#### 密钥状态说明

- 🟢 **活跃** - 可以正常使用
- 🔴 **已停用** - 无法使用（停用后会自动用占位图片）

#### 配额管理

| 信息 | 说明 |
|------|------|
| **已使用 / 总限额** | 如 `50 / 1000` 表示已使用50次，总共可用1000次 |
| **无限制** | 表示没有设置配额限制 |
| **过期时间** | 显示密钥会在什么时间失效 |

### 停用密钥

**步骤：**
1. 在"API密钥管理"页面找到要停用的密钥
2. 点击该密钥行的 **"停用"** 按钮
3. 确认停用
4. 密钥立即停用，新的生成请求将使用占位图片

## ⚙️ 模型配置管理

### 添加模型配置

**步骤：**
1. 在模型列表中找到目标模型
2. 点击 **"配置"** 按钮
3. 填写配置表单

#### 配置表单字段

| 字段 | 说明 | 示例 |
|------|------|------|
| **配置键** | 配置的唯一标识 | `pricing`, `limits`, `parameters` |
| **配置值** | JSON格式的配置内容 | `{"per_image": 0.02, "currency": "USD"}` |
| **描述** | 配置的说明文字 | `Model pricing information` |

### 常见配置示例

#### 1. 定价配置
```json
配置键: pricing
配置值: {
  "per_image": 0.02,
  "currency": "USD",
  "min_charge": 0.01
}
```

#### 2. 参数限制配置
```json
配置键: limits
配置值: {
  "max_batch": 10,
  "min_prompt_length": 10,
  "max_prompt_length": 1000,
  "timeout_seconds": 120
}
```

#### 3. 模型参数配置
```json
配置键: parameters
配置值: {
  "size": "1024x1024",
  "quality": "standard",
  "style": "natural"
}
```

## 🔗 不同提供商的配置指南

### OpenAI (DALL-E)

#### 获取API密钥
1. 访问 https://platform.openai.com/api-keys
2. 创建新的API密钥
3. 复制密钥内容（以 `sk-` 开头）

#### 推荐配置
```json
配置键: openai_config
配置值: {
  "model": "dall-e-3",
  "size": "1024x1024",
  "quality": "standard",
  "n": 1
}
```

#### 定价参考
```json
配置键: pricing
配置值: {
  "dall-e-3": {
    "1024x1024": 0.02,
    "1024x1792": 0.025,
    "1792x1024": 0.025
  },
  "dall-e-2": {
    "1024x1024": 0.020,
    "512x512": 0.018,
    "256x256": 0.016
  }
}
```

### Stability AI

#### 获取API密钥
1. 访问 https://platform.stability.ai/account/keys
2. 创建新的API密钥
3. 复制密钥内容

#### 推荐配置
```json
配置键: stability_config
配置值: {
  "engine_id": "stable-diffusion-xl-1024-v1-0",
  "steps": 20,
  "cfg_scale": 7.0
}
```

### Hugging Face

#### 获取API密钥
1. 访问 https://huggingface.co/settings/tokens
2. 创建新的API令牌
3. 复制令牌内容

#### 推荐配置
```json
配置键: hf_config
配置值: {
  "model_id": "stabilityai/stable-diffusion-2-1",
  "use_auth_token": true
}
```

## 📱 前端集成说明

### 模型选择下拉菜单

生成图片时，用户可以在BottomPanel中看到：
- 从数据库动态加载的模型列表
- 每个模型的提供商信息
- 模型详情（点击info图标）

### 自动API调用流程

```
用户选择模型 
    ↓
输入生成提示词
    ↓
点击生成按钮
    ↓
后端自动查询数据库
    ↓
获取模型对应的API密钥
    ↓
调用该提供商的API
    ↓
生成图片
```

### 如果API密钥不可用

系统会自动：
1. 检测API密钥无效或过期
2. 生成占位图片替代
3. 继续完成生成流程（用户体验不中断）

## 🛠️ 故障排查

### 问题1：页面加载缓慢

**原因**：后端未启动或数据库连接失败

**解决方案**：
```bash
# 检查后端是否运行
curl http://localhost:8000/api/models

# 如果无响应，启动后端
cd backend
python run.py
```

### 问题2：添加API密钥后仍然使用占位图

**原因**：
- API密钥已过期
- API密钥已达配额限制
- API密钥格式不正确

**解决方案**：
1. 检查密钥是否是最新的（从提供商重新获取）
2. 检查配额限制是否已满
3. 验证密钥格式是否正确

### 问题3：配置值无法保存

**原因**：JSON格式不正确

**解决方案**：
- 确保JSON格式有效
- 在线使用 JSON 验证工具：https://jsonlint.com/
- 例如 ✅ 正确：`{"key": "value"}`
- 例如 ❌ 错误：`{'key': 'value'}`（使用了单引号）

### 问题4：模型列表为空

**原因**：数据库中没有模型数据

**解决方案**：
```bash
# 检查数据库是否初始化
psql -U postgres -d picture

# 运行初始化脚本
psql -U postgres -d picture -f backend/app/migrations/add_model_management.sql
```

## 🔒 安全建议

1. **API密钥保护**
   - 不要在代码中硬编码API密钥
   - 通过页面或环境变量添加密钥
   - 定期轮换密钥

2. **配额管理**
   - 为每个密钥设置合理的配额限制
   - 监控使用情况
   - 及时发现异常

3. **过期管理**
   - 为密钥设置合理的过期时间
   - 定期更新过期的密钥
   - 提前规划替换

## 📞 常见问题

**Q: 如何重置API密钥？**
A: 停用旧密钥，然后添加新的API密钥。系统会自动使用最新的活跃密钥。

**Q: 一个模型可以配置多个API密钥吗？**
A: 可以。系统会自动选择活跃的密钥。这样做可以实现密钥轮换和故障转移。

**Q: 配置值支持什么格式？**
A: 仅支持有效的JSON格式。可以包含任何JSON数据结构。

**Q: 如何知道API密钥的配额已满？**
A: 查看"API密钥管理"页面，可以看到"已使用/总限额"的对比。

**Q: 停用密钥后会怎样？**
A: 新的生成请求将不使用该密钥，系统会自动选择其他活跃密钥，或使用占位图片。
