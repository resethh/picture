# AI图像生成平台技术实现方案

## 一、项目概述

基于 FastAPI + React + ReactFlow 构建的节点式 AI 图像生成平台。

**MVP 范围：** 纯图片生成模式，单用户本地部署，无需登录系统。

---

## 二、技术栈选型

### 2.1 后端技术栈

| 技术项 | 选型 | 说明 |
|--------|------|------|
| Web 框架 | FastAPI | 高性能异步框架，原生支持类型提示 |
| 运行时 | Uvicorn | ASGI 服务器 |
| 数据存储 | JSON 文件 | 不使用 ORM，直接用文件系统存储结构化数据 |
| 文件存储 | 本地磁盘 | `backend/media/` 目录，分 `uploads/` 和 `generated/` |
| 异步任务 | FastAPI BackgroundTasks + 内存队列 | 单机异步任务处理 |
| AI 模型 | 占位实现 | 先用假模型（延迟 + 占位图）打通流程 |

### 2.2 前端技术栈

| 技术项 | 选型 | 说明 |
|--------|------|------|
| 框架 | React 18 + TypeScript | 现代化组件开发 |
| 构建工具 | Vite | 极速开发体验 |
| 画布引擎 | ReactFlow | 专业的节点编辑器库 |
| UI 组件库 | Ant Design | 成熟的 React 组件库 |
| 状态管理 | Zustand | 轻量级状态管理 |
| HTTP 客户端 | Axios | 接口请求封装 |

### 3.2 数据流设计

**上传图片流程：**
```
用户点击 + 按钮 → 选择文件 → POST /api/images/upload 
→ 后端保存文件 → 返回 imageId 和 URL 
→ 前端在画布创建 ImageNode
```

**生成图片流程：**
```
用户输入 Prompt + 选择模型 + 点击执行按钮 
→ POST /api/generate 
→ 后端创建任务（返回 taskId） 
→ 后台任务开始执行 
→ 前端创建 GenerateNode 
→ 轮询 GET /api/generate/{taskId} 
→ 更新进度条 
→ 完成后显示生成的图片
```

**保存项目流程：**
```
用户点击保存按钮 
→ 收集画布状态（nodes, edges, viewport） 
→ POST /api/projects 
→ 后端生成 projectId 并保存 JSON 文件 
→ 返回 projectId
```

## 五、后端 API 设计

### 5.1 接口清单

| 方法 | 路径 | 功能 | 备注 |
|------|------|------|------|
| GET | `/` | 健康检查 | 返回 `{"status": "ok"}` |
| POST | `/api/images/upload` | 上传图片 | multipart/form-data |
| GET | `/media/{path}` | 静态文件访问 | 由 FastAPI StaticFiles 处理 |
| POST | `/api/generate` | 创建生成任务 | 返回 taskId |
| GET | `/api/generate/{taskId}` | 查询任务状态 | 返回进度和结果 |
| POST | `/api/projects` | 保存项目 | 返回 projectId |
| GET | `/api/projects/{projectId}` | 加载项目 | 返回项目完整数据 |

### 7.4 轮询机制

```typescript
const startPollingTask = (taskId: string) => {
  const interval = setInterval(async () => {
    const status = await generateAPI.getStatus(taskId);
    
    updateTaskProgress(taskId, status.progress);
    
    if (status.status === 'success') {
      updateNodeData(nodeId, { 
        status: 'success', 
        resultImages: status.resultImages 
      });
      clearInterval(interval);
    }
    
    if (status.status === 'failed') {
      updateNodeData(nodeId, { 
        status: 'failed', 
        error: status.error 
      });
      clearInterval(interval);
    }
  }, 1000); // 每秒轮询一次
};
```

## 十一、风险与注意事项

### 11.1 技术风险

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| JSON 文件并发写入冲突 | 数据丢失 | 实现文件锁机制 |
| 内存队列重启丢失 | 任务状态丢失 | 定期持久化到 JSON 文件 |
| 大文件上传超时 | 用户体验差 | 限制文件大小，添加上传进度 |
| ReactFlow 性能问题 | 节点过多时卡顿 | 限制画布节点数量，优化渲染 |

### 11.2 开发注意事项

1. **文件路径处理：** Windows 下注意路径分隔符（使用 `Path` 对象）
2. **CORS 配置：** 开发环境需配置跨域
3. **图片格式验证：** 严格校验上传文件类型
4. **错误处理：** 统一异常处理和错误响应格式
5. **日志记录：** 关键操作记录日志便于调试
