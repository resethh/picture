# PostgreSQL 数据库部署说明

## 功能说明

项目已集成 PostgreSQL 数据库持久化功能，支持**数据库**和**文件**两种存储方式自由切换。

### 技术特点
- ✅ 使用原生 SQL，不依赖 ORM
- ✅ 连接池管理，提升性能
- ✅ 支持环境变量配置
- ✅ 可在数据库/文件存储间切换

## 数据库表结构

### 1. projects（项目表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 项目ID（主键）|
| name | VARCHAR(255) | 项目名称 |
| graph | JSONB | 画布图数据 |
| thumbnail | TEXT | 缩略图URL |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间（自动更新）|

### 2. tasks（生成任务表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 任务ID（主键）|
| image_ids | TEXT[] | 输入图片ID数组 |
| mode | VARCHAR(50) | 生成模式 |
| model_name | VARCHAR(100) | AI模型名称 |
| prompt | TEXT | 提示词 |
| ratio | VARCHAR(20) | 生成比例 |
| count | INTEGER | 生成数量 |
| status | VARCHAR(20) | 任务状态 |
| progress | INTEGER | 任务进度(0-100) |
| result_images | JSONB | 生成结果图片 |
| error | TEXT | 错误信息 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间（自动更新）|

### 3. images（上传图片表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(50) | 图片ID（主键）|
| filename | VARCHAR(255) | 文件名 |
| url | TEXT | 访问URL |
| width | INTEGER | 图片宽度 |
| height | INTEGER | 图片高度 |
| file_size | INTEGER | 文件大小 |
| created_at | TIMESTAMP | 上传时间 |

## 快速部署

### 步骤 1：安装 PostgreSQL

**Windows:**
```bash
# 下载安装 PostgreSQL 16.x
https://www.postgresql.org/download/windows/

# 或使用 Chocolatey
choco install postgresql
```

**Mac:**
```bash
brew install postgresql@16
brew services start postgresql@16
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
```

### 步骤 2：创建数据库

```bash
# 登录 PostgreSQL
psql -U postgres

# 创建数据库
CREATE DATABASE picturev1;

# 退出
\q
```

### 步骤 3：配置环境变量

复制 `.env.example` 为 `.env` 并修改配置：

```bash
cd backend
cp .env.example .env
```

编辑 `.env` 文件：
```ini
# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_NAME=picturev1
DB_USER=postgres
DB_PASSWORD=你的密码
DB_MIN_CONN=1
DB_MAX_CONN=10

# 启用数据库（true=数据库，false=文件）
USE_DATABASE=true
```

### 步骤 4：安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

### 步骤 5：初始化数据库表

```bash
python -m app.init_database
```

执行成功会看到：
```
开始初始化数据库...
数据库配置: localhost:5432/picturev1
✅ 数据库表结构初始化成功！

📋 已创建的表：
  - projects
  - tasks
  - images
```

### 步骤 6：启动服务

```bash
python run.py
```

## 配置说明

### 使用数据库存储
在 `.env` 中设置：
```ini
USE_DATABASE=true
```

### 使用文件存储（原方式）
在 `.env` 中设置：
```ini
USE_DATABASE=false
```

### 数据库连接池配置
```ini
DB_MIN_CONN=1    # 最小连接数
DB_MAX_CONN=10   # 最大连接数
```

## 常见问题

### 1. 连接失败
**问题:** `❌ 数据库连接失败: FATAL: password authentication failed`

**解决:**
- 检查 `.env` 中的用户名和密码
- 确认 PostgreSQL 服务已启动
- 检查 `pg_hba.conf` 配置允许本地连接

### 2. 表初始化失败
**问题:** 初始化脚本执行出错

**解决:**
```bash
# 重新执行初始化
python -m app.init_database
```

### 3. psycopg2 安装失败
**问题:** Windows 上安装 psycopg2-binary 失败

**解决:**
```bash
# 使用预编译版本
pip install psycopg2-binary --only-binary :all:
```

### 4. 切换存储方式
数据库和文件存储的数据不会自动同步，切换时需要手动迁移数据或重新开始。

## 性能优化建议

1. **连接池调优**
   - 根据并发量调整 `DB_MAX_CONN`
   - 建议值：并发用户数 × 2

2. **索引优化**
   - 已自动创建常用字段索引
   - 按更新时间排序已优化

3. **查询优化**
   - 使用 JSONB 类型高效存储 JSON
   - 使用连接池避免频繁连接

## 数据迁移

### 从文件迁移到数据库

可以编写迁移脚本读取 `backend/data/projects/*.json` 和 `backend/data/tasks/*.json`，调用数据库服务层的保存方法。

示例：
```python
from pathlib import Path
from app.services.storage import load_json
from app.services.db_storage import save_project_to_db

projects_dir = Path("backend/data/projects")
for file_path in projects_dir.glob("*.json"):
    data = load_json(file_path)
    if data:
        save_project_to_db(data["id"], data)
```

## 备份与恢复

### 备份数据库
```bash
pg_dump -U postgres picturev1 > backup.sql
```

### 恢复数据库
```bash
psql -U postgres picturev1 < backup.sql
```
