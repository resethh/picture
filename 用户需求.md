
AI图像生成工具技术文档
后端要求：fastapi

1. 概述

本工具是一个基于Web的AI图像生成平台，核心功能是通过上传参考图片和输入文本提示（Prompt），利用AI模型生成新的图像。其工作流采用“节点式”或“画布式”设计，允许用户清晰地看到数据流向和处理过程。

2. 网页布局 (UI Layout)

整个界面采用深色主题，背景为带有网格点的黑色画布，便于用户在无限空间中拖拽和组织节点。

2.1 顶部导航栏
- 左上角: 应用Logo/名称（图中为“优勤AI设计工作台”）。
- 中间: 当前项目名称（默认为“未命名”）。
- 右上角: 
  - “分享”按钮：用于分享当前项目。
  - “保存”按钮：用于保存当前项目状态。
    
2.2 主工作区 (Canvas)
- 核心区域: 占据屏幕绝大部分，是用户进行操作的主要画布。
- 节点 (Nodes): 所有操作单元都以矩形框的形式呈现，例如“图片”节点、“图片生成”节点等。
- 连接线 (Edges): 节点之间通过绿色或白色的曲线连接，表示数据的输入输出关系，形成一个可视化的处理流程图。
  
2.3 底部控制面板 (Bottom Panel)
该面板位于屏幕底部，是一个深色、圆角矩形的工具栏，集成了所有启动和配置AI生成任务的关键控件。
1. 核心输入框
- 位置: 面板中央。
- 功能: 这是用户输入文本提示（Prompt）的主要区域。
- 交互:
  - 用户可以在此处直接键入对目标图像或视频的文字描述。
  - 当选择不同的生成模式（如“图片生成”或“视频生成”）时，输入框内的占位符文字会动态变化，以提供相应的引导信息。
    - 图片生成模式: 提示语为“请输入你想更生成的画面描述”。
    - 视频生成模式: 提示语变为“请描述你想更生成的视频（如果是通过图片生成视频：SR2 Pro模型要求，图片中不能有人物）”，这表明系统会根据所选模型提供上下文相关的提示。

2. 添加素材按钮
- 位置: 输入框左侧，一个带“+”号的圆形按钮。
- 功能: 用于上传参考图片或视频作为生成的依据。
- 交互: 点击后会弹出系统的文件选择器，允许用户从本地磁盘或云存储中选择文件。
  
3. AI模型与模式选择器
- 位置: 输入框左下方。
- 功能: 用于切换AI生成的类型（图片/视频）和具体使用的AI模型。
- 交互:
  - 点击此区域会弹出一个下拉菜单。
  - 菜单内分为两个主要标签页：“图片生成”和“视频生成”。
  - 在每个标签页下，列出了一系列可用的AI模型供用户选择，例如：
    - 图片生成: 全能图片 Pro, 全能图片, Seedream 4.5, Image1 等。
    - 视频生成: SR2, SR2 Pro, Vo 3.1, Hailuo 2.3, Xling 2.5 等。
  - 用户点击模型名称旁边的复选框即可选中。被选中的模型会高亮显示，并且其名称会同步显示在面板上。
  - 选择不同模型后，系统会自动调整输入框的提示语和右侧的参数选项，以匹配该模型的要求。
    
4. 生成参数设置区
- 位置: 面板右侧。
- 功能: 根据当前选择的AI模型，提供一系列可调节的生成参数。
- 交互:
  - 分辨率/画布大小: 显示为 1K 或 9:16 等，点击下拉箭头可选择不同的尺寸比例。
  - 生成数量: 显示为“1张”，点击下拉箭头可选择一次生成多张图片（如2张、3张、4张），以增加选择范围。
  - 视频时长: 当选择视频生成模型（如 SR2 Pro）时，此处会显示视频时长选项（如 4s, 8s），并可进行切换。
  - 其他参数: 可能包含帧率（如 720P）、随机种子等高级设置，但在此视频中未完全展示。
    
5. 执行按钮 (绿色圆形按钮)
- 位置: 面板最右侧。
- 功能: 这是启动AI生成任务的最终确认按钮。
- 交互:
  - 当用户完成所有配置（包括输入Prompt、选择模型和参数）后，点击此按钮。
  - 点击后，系统会在主工作区（Canvas）创建一个新的“图片生成”或“视频生成”节点，并开始执行生成任务，节点内会显示进度条。
    


2.4 右侧工具栏 (Right Sidebar)
1. 平移模式 (Pan Mode)
- 图标: 一个手形图标。
- 功能: 启用后，用户可以按住鼠标左键在画布上拖拽，从而自由移动整个画布的视角，方便浏览画布的不同区域。
- 交互: 点击此按钮后，光标会变为手形，表示当前处于平移状态。再次点击或选择其他工具可退出该模式。
2. 框选模式 (Select Mode)
- 图标: 一个带虚线框的图标（通常表示选择或框选）。
- 功能: 启用后，用户可以通过鼠标拖拽绘制一个矩形框来选择画布上的一个或多个节点。
- 交互: 点击此按钮后，用户可以在画布上点击并拖动以创建一个选择框，框内的所有节点都会被选中，之后可以对它们进行统一操作（如移动、删除等）。
3. 放大视图 (Zoom In)
- 图标: 加号 + 图标。
- 功能: 放大画布的显示比例，让用户可以更清晰地查看节点的细节、文字和连接线。
- 交互: 每点击一次，画布会以预设的比例（例如10%）进行放大。
4. 缩小视图 (Zoom Out)
- 图标: 减号 - 图标。
- 功能: 缩小画布的显示比例，让用户能获得更广阔的视野，以便总览整个工作流的布局和结构。
- 交互: 每点击一次，画布会以预设的比例（例如10%）进行缩小。
5. 自适应视图 (Fit View)
- 图标: 一个方框内含四个向外箭头的图标。
- 功能: 将画布自动缩放并居中，使其内容完全适配当前窗口大小，确保所有节点都可见。
- 交互: 点击后，无论当前画布被放大、缩小或平移到了哪个位置，都会立即调整到最佳的“全览”状态。
6. 切换视图 (Toggle View / Preview)
- 图标: 一个眼睛形状的图标。
- 功能: 此功能可能用于切换不同的工作视图或预览模式。例如，在“编辑模式”和“预览模式”之间切换，或者隐藏/显示某些辅助元素（如网格点、连接线等）。
- 交互: 点击后，界面会切换到另一种显示状态，以满足不同的操作需求。
3. 核心功能与操作流程
3.1 功能模块
3.1.1 图片上传 (Image Upload)
- 入口: 用户双击画布空白位置或底部面板的“+”号按钮可添加图片节点。
- 操作: 点击节点内的“选择文件”或“上传”按钮，会弹出系统文件选择器。
- 效果: 选中的图片会显示在节点内，并标注其尺寸（如 768 x 1280）。多个图片可以被上传并放置在画布的不同位置。
  
3.1.2 AI模型选择 (Model Selection)
- 入口: 点击底部面板的下拉菜单。
- 操作: 展开一个列表，列出所有可用的AI模型，如 SDXL Pro, Stable Diffusion 4.5, Midjourney 2.0 等。
- 效果: 用户勾选一个模型后，该模型将被应用于后续的图像生成任务。
  
3.1.3 文本提示输入 (Prompt Input)
- 入口: 底部中央的文本输入框。
- 操作: 用户在此处输入对目标图像的文字描述，例如“小一点的房间”。
- 效果: 输入的文本会被作为指令发送给AI模型，指导其生成符合描述的图像。
  
3.1.4 图像生成 (Image Generation)
- 入口: 在配置好图片、模型和Prompt后，点击底部面板的“生成”按钮。
- 操作: 系统会创建一个新的“图片生成”节点，并自动将其与上游的图片节点和Prompt输入连接起来。
- 效果: 
  - 节点内显示“生成中...XX%”的进度条。
  - 生成完成后，节点下方会显示生成的图片缩略图。
  - 生成的图片可以被拖动到画布其他位置，或作为新的输入源连接到其他节点。
    
3.2 完整操作流程示例
1. 准备素材: 用户从本地文件夹选择并上传两张参考图片，它们分别出现在画布上的两个“图片”节点中。
2. 配置任务: 
  - 用户点击底部模型选择下拉菜单，选定 SDXL Pro 模型。
  - 用户在文本输入框中输入Prompt：“小一点的房间”。
3. 执行生成: 用户点击“生成”按钮。
4. 查看结果: 
  - 系统在画布中央创建一个“图片生成”节点。
  - 该节点自动连接到两个图片节点和Prompt输入。
  - 节点开始显示“生成中...10%”的进度。
  - 生成完成后，节点下方出现一张或多张生成的图片预览。
5. 后续操作: 用户可以继续编辑Prompt、更换模型或上传更多图片，重复上述步骤进行迭代创作。
  
4. 开发指导
4.1 前端开发
- 框架: 推荐使用支持复杂交互和图形渲染的现代前端框架，如 React + TypeScript。
- 画布组件: 需要实现一个可拖拽、缩放、连接节点的画布组件。可以考虑使用开源库如 react-flow 或 dagre-d3。
- 节点组件: 为“图片”、“图片生成”等不同类型节点创建独立的React组件，管理各自的UI和状态。
- 状态管理: 使用 Redux 或 Context API 管理全局状态，如当前项目、已上传的图片、选中的模型、Prompt文本等。
- 文件上传: 实现与系统文件选择器的交互，并将图片文件读取为Base64或URL供前端显示。
  
4.2 后端开发
- API接口: 提供以下核心API：
  - POST /api/images/upload: 上传图片，返回图片ID和访问URL。
  - POST /api/generate: 接收图片ID列表、模型名称、Prompt文本，启动异步图像生成任务，返回任务ID。
  - GET /api/generate/{taskId}: 查询指定任务的状态和结果。
- AI模型集成: 后端需要集成或调用第三方AI模型服务（如Stable Diffusion API），将前端传来的参数转发给模型，并接收生成结果。
- 任务队列: 由于图像生成是耗时操作，必须使用消息队列（如 Redis Queue）来异步处理生成请求，避免阻塞主线程。
- 存储: 存储用户上传的原始图片和AI生成的图片。

4.3 数据流设计
- 整个应用的数据流应围绕“节点-连接”的概念构建。
- 每个节点都是一个数据处理单元，拥有自己的输入和输出。
- 连接线定义了数据的依赖关系，当上游节点数据更新时，下游节点应被触发重新计算或生成。
- 前端与后端的通信应遵循RESTful规范，确保状态同步。
